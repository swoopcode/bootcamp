HTML

HyperText Markup Language 또는 HTML 은 웹 브라우저 에 표시되도록 설계된 문서 의 표준 마크업 언어 입니다 . CSS( Cascading Style Sheets )와 같은 기술과 JavaScript 와 같은 스크립팅 언어 의 도움을 받을 수 있습니다.

웹 브라우저 는 웹 서버 또는 로컬 저장소에서 HTML 문서를 수신 하고 문서를 멀티미디어 웹 페이지로 렌더링 합니다. HTML은 의미론적 으로 웹 페이지 의 구조를 설명 하고 원래 문서의 모양에 대한 단서를 포함했습니다.

HTML 요소 는 HTML 페이지의 빌딩 블록입니다. HTML 구성을 사용하면 이미지 및 대화형 양식 과 같은 기타 개체 를 렌더링된 페이지에 포함할 수 있습니다. HTML은 제목, 단락, 목록, 링크 , 따옴표 및 기타 항목과 같은 텍스트의 구조적 의미 를 표시하여 구조화된 문서 를 만드는 수단을 제공합니다 . HTML 요소는 꺾쇠 괄호 를 사용하여 작성된 태그 로 구분됩니다 . 와 같은 태그 는 콘텐츠를 페이지 에 직접 소개합니다. 다음과 같은 기타 태그<img /><input /><p>문서 텍스트에 대한 정보를 제공하고 다른 태그를 하위 요소로 포함할 수 있습니다. 브라우저는 HTML 태그를 표시하지 않지만 이를 사용하여 페이지의 내용을 해석합니다.

HTML은 JavaScript 와 같은 스크립팅 언어로 작성된 프로그램을 포함할 수 있으며 , 이는 웹 페이지의 동작과 콘텐츠에 영향을 미칩니다. CSS를 포함하면 콘텐츠의 모양과 레이아웃이 정의됩니다. HTML 의 이전 관리자이자 현재 CSS 표준의 관리자인 W3C( World Wide Web Consortium )는 1997년부터 명시적인 프레젠테이션 HTML보다 CSS의 사용을 권장했습니다. [2] HTML5 로 알려진 HTML 형식은 다음과 같은 용도로 사용됩니다. 주로 요소를 사용하여 JavaScript와 공동으로 비디오 및 오디오를 표시합니다.

역사

개발

1980년 에 CERN 의 계약자 인 물리학자 Tim Berners-Lee 는 CERN 연구원들이 문서를 사용하고 공유할 수 있는 시스템인 INQUIRE 를 제안하고 프로토타입 을 만들었습니다. 1989년 Berners-Lee는 인터넷 기반 하이퍼텍스트 시스템 을 제안하는 메모를 작성했습니다 . [3] Berners-Lee는 1990년 후반에 HTML을 지정하고 브라우저 및 서버 소프트웨어를 작성했습니다. 그해 Berners-Lee와 CERN 데이터 시스템 엔지니어인 Robert Cailliau 는 공동 자금 요청에 협력했지만 이 프로젝트는 CERN에서 공식적으로 채택되지 않았습니다. 1990년 의 개인 노트 [4] 에서 그는 [5] 를 나열했습니다 ."하이퍼텍스트가 사용되는 많은 영역 중 일부" 그리고 백과사전 을 먼저 두십시오.

공개적으로 사용 가능한 HTML에 대한 최초의 설명은 "HTML 태그" 라는 문서 로 1991년 후반에 Tim Berners-Lee가 인터넷에서 처음 언급했습니다. [6] [7] HTML의 초기 비교적 단순한 디자인을 구성하는 18개의 요소를 설명합니다. 하이퍼링크 태그를 제외하고 이들은 CERN 의 사내 SGML( Standard Generalized Markup Language ) 기반 문서 형식 인 SGMLguid 의 영향을 많이 받았습니다 . 이러한 요소 중 11개는 여전히 HTML 4에 존재합니다.

HTML은 웹 브라우저 가 텍스트, 이미지 및 기타 자료를 시각적 또는 청각적 웹 페이지 로 해석하고 구성 하는 데 사용하는 마크업 언어 입니다 . HTML 마크업의 모든 항목에 대한 기본 특성은 브라우저에서 정의되며 이러한 특성은 웹 페이지 디자이너가 CSS 를 추가로 사용하여 변경하거나 향상시킬 수 있습니다 . 텍스트 요소의 대부분은 1988년 ISO 기술 보고서 ​​TR 9537 SGML을 사용하기 위한 기술에서 찾을 수 있으며, 이는 다시 CTSS 용으로 1960년대 초에 개발된 RUNOFF 명령 에서 사용된 것과 같은 초기 텍스트 서식 지정 언어의 기능을 다룹니다.(Compatible Time-Sharing System) 운영 체제: 이러한 형식 지정 명령은 식자공이 수동으로 문서 형식을 지정하는 데 사용하는 명령에서 파생되었습니다. 그러나 일반화된 마크업의 SGML 개념은 구조와 마크업의 분리와 함께 단순한 인쇄 효과가 아니라 요소(속성이 있는 중첩된 주석 범위)를 기반으로 합니다. HTML은 CSS와 함께 이 방향으로 점진적으로 이동했습니다.

Berners-Lee는 HTML을 SGML의 응용 프로그램으로 간주했습니다. 1993년 중반에 Berners-Lee와 Dan Connolly 가 작성한 "Hypertext Markup Language(HTML)" Internet Draft인 HTML 사양에 대한 첫 번째 제안을 출판하면서 IETF( Internet Engineering Task Force ) 에 의해 공식적으로 정의되었습니다 . 문법을 정의하기 위한 SGML 문서 유형 정의 가 포함되었습니다. [9] [10] 초안은 6개월 후에 만료되었지만 성공적인 프로토타입에 기반한 표준에 대한 IETF의 철학을 반영하여 인라인 이미지를 삽입하기 위한 NCSA Mosaic 브라우저의 사용자 지정 태그를 인정한 것으로 유명합니다. 마찬가지로 Dave Raggett경쟁하는 Internet-Draft인 "HTML+(Hypertext Markup Format)"는 1993년 후반부터 테이블 및 채우기 양식과 같은 이미 구현된 기능을 표준화할 것을 제안했습니다.

HTML 및 HTML+ 초안이 1994년 초에 만료된 후 IETF는 HTML 작업 그룹을 만들었고 1995년에 "HTML 2.0"을 완성했습니다.

IETF의 후원 아래 추가 개발은 경쟁 이해 관계로 인해 중단되었습니다. 1996년부터 HTML 사양은 W3C( World Wide Web Consortium ) 에서 상용 소프트웨어 공급업체의 의견을 받아 유지 관리해 왔습니다 . [13] 그러나 2000년에 HTML도 국제 표준이 되었습니다( ISO / IEC 15445:2000). HTML 4.01은 1999년 후반에 발표되었으며 추가 정오표는 2001년까지 발표되었습니다. 2004년 WHATWG( Web Hypertext Application Technology Working Group )에서 HTML5에 대한 개발이 시작되어 2008년에 W3C와 공동 결과물이 되었으며 완료 및 표준화되었습니다. 2014년 10월 28일.

HTML 버전 타임라인

HTML2

1995년 11월 24일

HTML 2.0은 RFC 1866 으로 게시되었습니다 . 추가 RFC 추가 기능:

  • 1995년 11월 25일: RFC 1867 (양식 기반 파일 업로드)
  • 1996년 5월: RFC 1942 (표)
  • 1996년 8월: RFC 1980 (클라이언트측 이미지 맵)
  • 1997년 1월: RFC 2070 ( 국제화 )

HTML3

1997년 1월 14일

HTML 3.2 [15] 는 W3C 권장 사항 으로 게시되었습니다 . IETF가 1996년 9월 12일에 HTML 작업 그룹을 종료했기 때문에 W3C에서 독점적으로 개발하고 표준화한 첫 번째 버전이었습니다. [16]

처음에 코드명 "Wilbur"인 [17] HTML 3.2는 수학 공식을 완전히 삭제하고 다양한 독점 확장 간의 중복을 조정했으며 Netscape 의 시각적 마크업 태그 대부분을 채택했습니다. Netscape의 blink 요소 와 Microsoft 의 marquee 요소 는 두 회사 간의 상호 합의로 인해 생략되었습니다. [13] HTML과 유사한 수학 공식에 대한 마크업은 14개월 후 MathML 에서 표준화되었습니다 .

HTML4

1997년 12월 18일

HTML 4.0 [18] 은 W3C 권장 사항으로 게시되었습니다. 세 가지 변형을 제공합니다.

  • 더 이상 사용되지 않는 요소가 금지되는 Strict
  • 더 이상 사용되지 않는 요소가 허용되는 전환
  • 대부분 프레임 관련 요소만 허용되는 프레임셋.

처음에 코드명 "Cougar"인 [17] HTML 4.0은 많은 브라우저별 요소 유형과 속성을 채택했지만 동시에 Netscape의 시각적 마크업 기능을 더 이상 사용되지 않는 것으로 표시하고 스타일 시트를 선호하여 단계적으로 제거하려고 했습니다. HTML 4는 ISO 8879 – SGML을 준수하는 SGML 응용 프로그램입니다.

1998년 4월 24일

HTML 4.0 [20] 은 버전 번호를 올리지 않고 약간만 수정하여 재발행되었습니다.

1999년 12월 24일

HTML 4.01 [21] 은 W3C 권장 사항으로 게시되었습니다. HTML 4.0과 동일한 세 가지 변형을 제공하며 마지막 정오표 는 2001년 5월 12일에 게시되었습니다.

2000년 5월

ISO/IEC 15445:2000 [22] [23] (" ISO HTML", HTML 4.01 Strict 기반)이 ISO/IEC 국제 표준으로 발표되었습니다. ISO에서 이 표준은 ISO/IEC JTC 1/SC 34 (ISO/IEC 공동 기술 위원회 1, 소위원회 34 – 문서 설명 및 처리 언어) 의 영역에 속합니다 . [22] HTML 4.01 이후에는 병렬 XML 기반 언어 XHTML의 개발이 2000년대 초반과 중반까지 W3C의 HTML 워킹 그룹을 차지하면서 수년 동안 새로운 HTML 버전이 없었습니다.
